# UI/UX 设计标准

## 视觉设计
- **颜色系统**：使用 Tailwind CSS 内置变量（`bg-primary`、`text-primary-foreground`、`bg-background`）
- **颜色限制**：除非明确要求，否则不得使用靛蓝或蓝色
- **主题 支持**：使用下一个主题实现亮/暗模式
- **版式**：具有适当字体粗细和大小的一致层次结构

## 响应式设计（强制）
- **移动优先**：针对移动设备进行设计，然后针对桌面设备进行增强
- **断点**：使用 Tailwind 响应式前缀（`sm:`、`md:`、`lg:`、`xl:`）
- **触摸友好**：交互式元素的最小触摸目标为 44 像素

## 辅助功能（强制）
- **语义 HTML**：使用 `main`、`header`、`nav`、`section`、`article`
- **ARIA 支持**：正确的角色、标签和描述
- **屏幕阅读器**：使用 `sr-only` 类来获取屏幕阅读器内容
- **替代文本**：所有图像的描述性替代文本
- **键盘导航**：确保所有元素均可通过键盘访问

## 互动元素
- **加载状态**：在异步操作期间显示旋转器/骨架
- **错误 处理**：清晰、可操作的 错误 消息
- **反馈**：用户 操作的 Toast 通知
- **动画**：微妙的 Framer 运动过渡（悬停、焦点、页面过渡）
- **悬停效果**：所有可点击元素上的交互式 反馈
